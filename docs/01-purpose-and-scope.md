# Purpose And Scope

## Purpose

`invoice-auto` provides a desktop interface for two related finance workflows:

1. Invoice generation from client, service, and invoice-number workbooks.
2. Email-agenda generation from the generated accountability CSV, pending-invoice data, and client communication settings.

The application is intended to reduce repeated manual work while preserving the finance team's existing Excel-based data process.

## In Scope

- Selecting Excel, CSV, and folder inputs through the desktop UI.
- Loading clients from the client master for selection.
- Generating monthly invoices in English or Spanish.
- Generating credit notes when the configured client record requires one.
- Generating or extending the QuickBooks accountability CSV.
- Generating `INVOICE_SUMMARY.xlsx` and `EMAIL_AGENDA.xlsx`.
- Creating email content and attachment-name lists for later use by the finance process.
- Packaging the application as a Windows installer.

## Out Of Scope

- Sending emails directly.
- Uploading invoices to a remote accounting or document-management system.
- Maintaining the authoritative client, service, pending-invoice, or invoice-master data.
- Managing user accounts or application authentication.
- Providing a database or server API.
- Replacing finance approval of invoice content, totals, recipients, or payment information.

## User Workflows

### Invoice Generation

The operator selects four inputs:

| UI field | Meaning |
| --- | --- |
| Client Master | Client and bank metadata used for invoice content and client selection |
| Services per Client | Workbook containing the `Client Data` sheet and service rows |
| Invoice Number Master | Invoice numbering and client-specific invoice metadata |
| Output Folder | Parent directory for generated files |

The operator selects a generation mode, date, and clients, then starts generation. The application passes positional arguments to `scripts/main.py` and displays progress received from Electron.

### Email Agenda Generation

The operator selects:

| UI field | Meaning |
| --- | --- |
| Client Master | Client identity, language, recipient, and communication settings |
| Invoice Number Master | Invoice-month metadata used for email content |
| New Invoices CSV | Usually the accountability CSV generated by the invoice workflow |
| Pending Invoices Excel | Existing pending-invoice data used in email tables |
| Output Folder | Destination for `INVOICE_SUMMARY.xlsx` and `EMAIL_AGENDA.xlsx` |

The operator selects a mail date and hour. The application passes these values to `scripts/mail-agenda-generation.py`.

## Terminology

| Term | Meaning |
| --- | --- |
| Client Master | Excel workbook containing client, bank, language, payment, and communication metadata |
| Services per Client | Excel workbook whose `Client Data` sheet contains service rows by invoice ID |
| Invoice Number Master | Excel workbook containing invoice IDs, prefixes, number seeds, month seeds, and additional-invoice counts |
| Batch | Normal monthly invoice generation mode, represented internally by `gen_mode = 1` |
| Additional Invoice | Additional-invoice mode, represented internally by `gen_mode = 2` |
| Accountability CSV | QuickBooks-oriented CSV generated from invoice service rows |
| Email agenda | Excel workbook containing prepared email recipients, content, dates, and attachment names |
| Invoice ID | Internal client identifier, such as a sanitized placeholder `CLIENT01` |
| Bank Code | Grouping code used in display names and email-agenda grouping |
